What is Java Platform Independence? HOT

🤔 What is it?
Platform independence means you write your Java code ONCE, compile it, and that compiled file (bytecode) can run on ANY operating system - Windows, Mac, Linux - without changing a single line. The JVM (Java Virtual Machine) on each OS reads the same bytecode and translates it for that specific OS.

Think of it like a universal remote control. You have ONE remote (bytecode), and it works with any TV brand (operating system) because each TV has a receiver (JVM) that understands the same signal.
💡 Why does this exist?
Before Java (in C/C++ days), if you wrote code on Windows, you had to recompile it separately for Linux and Mac. This was painful. Java solved it by introducing an intermediate layer - the JVM.

Without Java (C/C++)With Java
Write code → Compile for Windows → Runs on Windows ONLYWrite code → Compile to bytecode → Runs on ANY OS with JVM
Need separate binaries for each OSOne .class file works everywhere
⏰ When is this important?
- When building server-side apps deployed on Linux but developed on Windows/Mac
- When building cross-platform desktop apps
- When distributing libraries (JARs) that need to work everywhere
Your Code (.java) | v javac compiler | v Bytecode (.class) <--- Platform INDEPENDENT | +----+----+----+ | | | | v v v v JVM JVM JVM JVM <--- Platform DEPENDENT (Win)(Mac)(Lin)(Sol) | | | | v v v v Native Machine Code
⚠️ Common Mistakes
❌ Saying "Java is platform independent" - WRONG! Java LANGUAGE is platform independent. The JVM itself IS platform dependent (you need a different JVM for each OS).

❌ Confusing .java file with .class file - The .java is source code, the .class is bytecode. Only bytecode is portable.
✅ Best Practice Interview Answer
"Java achieves platform independence through bytecode. The source code is compiled by javac into platform-independent bytecode (.class files). This bytecode is then interpreted or JIT-compiled by the platform-specific JVM into native machine code. So the motto is 'Write Once, Run Anywhere' - but the JVM itself is platform-dependent."

What is Bytecode? How is it different from Machine Code?

🤔 What is it?
Bytecode is the INTERMEDIATE code that javac produces. It's NOT human-readable source code, and it's NOT machine code that your CPU understands directly. It sits in between.

Analogy: Imagine you write a letter in English (source code). A translator converts it to a universal language like Esperanto (bytecode). Then each country's local translator (JVM) converts Esperanto to their local language (machine code). 
// Source code (Hello.java)
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}

// After: javac Hello.java -> Hello.class (bytecode)
// View with: javap -c Hello.class
//   0: getstatic     #2  // Field java/lang/System.out
//   3: ldc           #3  // String Hello
//   5: invokevirtual #4  // Method println
//   8: return

Human readable?YesNo (but javap can show it)No
Platform independent?YesYesNo
Executed byNothing directlyJVMCPU directly

Mention that every .class file starts with the magic number 0xCAFEBABE - this is how JVM verifies it's a valid class file. Fun fact interviewers love!

What is JIT (Just-In-Time) Compiler? HOT

🤔 What is it?
JIT is a component of the JVM Execution Engine that compiles frequently executed bytecode (called "hot spots") into native machine code at runtime. This makes Java programs faster because native code runs directly on CPU without interpretation.

Analogy: Imagine you translate a foreign recipe every time you cook (interpreter). But for your favorite dish that you cook daily, you memorize the translation (JIT). Now you cook faster because you don't need to translate each time.
💡 Why does it exist?
Without JIT (Interpreter only)With JIT
Every line translated every timeHot code compiled once, reused as native
Slow for loops, repeated methodsFast for hot paths
Low startup timeSlightly higher startup, much better throughput
Method called 1st time -> Interpreter runs it Method called 2nd time -> Interpreter runs it ... Method called 10,000th time -> JIT says "This is HOT!" | v Compiles to NATIVE CODE | v Future calls use native code (FAST!)
✅ Advanced Points for 6+ Yrs
- JIT has two compilers: C1 (Client) = fast compile, less optimization. C2 (Server) = slower compile, deep optimization.
- Tiered Compilation (default since Java 8): Uses C1 first for quick results, then C2 for hot methods.
- JIT does: Method inlining, dead code elimination, loop unrolling, escape analysis.
- Use -XX:+PrintCompilation to see what JIT is compiling.

Explain the ClassLoader Subsystem MUST KNOW

🤔 What is it?
The ClassLoader is responsible for finding .class files and loading them into JVM memory. It works on a principle called Delegation Hierarchy - before loading a class, it asks its parent first. If the parent can't find it, then it tries itself.

Analogy: A student (App ClassLoader) has a question. Before answering, they ask their teacher (Extension ClassLoader). The teacher asks the principal (Bootstrap ClassLoader). If the principal knows, great. If not, it goes back down the chain.
Bootstrap ClassLoader (C/C++ code) Loads: java.lang.*, java.util.* (core Java) Location: jrt:/java.base (Java 9+) | v Platform ClassLoader (was Extension) Loads: javax.*, extension JARs Location: $JAVA_HOME/lib/ext (pre-9) | v Application ClassLoader (System) Loads: YOUR code, classpath JARs Location: -classpath / -cp | v Custom ClassLoaders (optional) Use case: App servers, plugin systems
💡 Why Delegation Model?
Security! If someone creates a malicious java.lang.String class, the Bootstrap ClassLoader will always load the REAL String class first. The malicious one never gets loaded. This is called the Sandbox Security Model. 
// See which classloader loaded a class
System.out.println(String.class.getClassLoader());
// Output: null (Bootstrap - implemented in native code)

System.out.println(MyApp.class.getClassLoader());
// Output: jdk.internal.loader.ClassLoaders$AppClassLoader

What is the static keyword? Explain ALL uses. HOT

🤔 What is it?
static means "belongs to the CLASS, not to any specific object." When something is static, there's only ONE copy shared across ALL objects of that class.

Analogy: In an apartment building, each flat has its own kitchen (instance variable). But there's ONE shared swimming pool (static variable) that all residents use.
5 Uses of static:
// 1. Static Variable (Class Variable)
public class Employee {
    static int companyId = 100;   // ONE copy shared by ALL employees
    String name;                    // Each employee has their OWN name
}

// 2. Static Method
public static int add(int a, int b) {
    return a + b;    // No need to create object. Call: Math.add(5,3)
}

// 3. Static Block (runs once when class is loaded)
static {
    System.out.println("I run ONCE when class is first loaded!");
    // Use: Loading config, initializing DB connections, loading native libs
}

// 4. Static Nested Class
public class Outer {
    static class Inner {
        // Can be created WITHOUT outer instance: new Outer.Inner()
    }
}

// 5. Static Import
import static java.lang.Math.PI;
import static java.lang.Math.sqrt;
// Now use PI directly instead of Math.PI
⚠️ Common Mistakes
❌ Accessing instance variable from static method: static method can't access non-static members directly (because there's no object context!)
❌ Overriding static methods: You CAN'T override static methods. You can only HIDE them. This is called Method Hiding.
❌ Memory leak with static: Static variables live as long as the class is loaded (essentially forever). Don't store large objects in static fields carelessly.

What is the final keyword? Explain ALL uses.

🤔 What is it?
final means "this CANNOT be changed/modified/overridden." It's Java's way of making things immutable or locked down. 
// 1. final Variable = CONSTANT (can't reassign)
final int MAX = 100;
MAX = 200; // COMPILE ERROR!

// 2. final Method = CAN'T be overridden by subclass
class Parent {
    final void doSomething() {
        // Child class CANNOT override this method
    }
}

// 3. final Class = CAN'T be extended/inherited
final class ImmutableConfig {
    // NO class can extend this. String, Integer are final classes!
}

// 4. final Parameter = can't reassign inside method
void process(final String name) {
    name = "other"; // COMPILE ERROR!
}

// 5. Blank final variable (assigned in constructor)
class User {
    final String id;  // No value here...
    User(String id) {
        this.id = id;    // ...assigned here. MUST be assigned in constructor!
    }
}
⚠️ TRICKY: final does NOT mean the OBJECT is immutable!
final List<String> list = new ArrayList<>();
list.add("Hello");     // This WORKS! You can modify the object.
list = new ArrayList<>(); // This FAILS! Can't reassign the reference.
// final locks the REFERENCE, not the OBJECT it points to!

What is Autoboxing and Unboxing? HOT

🤔 What is it?
Autoboxing = Automatic conversion of primitive to Wrapper object (int -> Integer)
Unboxing = Automatic conversion of Wrapper object to primitive (Integer -> int)

Java does this for you so you don't have to manually convert. 
// AUTOBOXING: primitive -> Wrapper
int num = 5;
Integer obj = num;  // Compiler does: Integer.valueOf(5)

// UNBOXING: Wrapper -> primitive
Integer obj2 = new Integer(10);
int num2 = obj2;   // Compiler does: obj2.intValue()

// Used everywhere with Collections (can't store primitives)
List<Integer> list = new ArrayList<>();
list.add(42);       // Autoboxing: 42 -> Integer.valueOf(42)
int val = list.get(0); // Unboxing: Integer -> int
⚠️ TRICKY Integer Cache Interview Question!
Integer a = 127;
Integer b = 127;
System.out.println(a == b);  // TRUE! (cached: -128 to 127)

Integer c = 128;
Integer d = 128;
System.out.println(c == d);  // FALSE! (not cached, new objects)
System.out.println(c.equals(d));  // TRUE! (compares value)
Java caches Integer objects from -128 to 127 for performance. Always use .equals() for wrapper comparisons!
⚠️ NullPointerException Trap
Integer x = null;
int y = x;  // BOOM! NullPointerException during unboxing!
// Always null-check wrapper types before unboxing

Type Casting: Widening vs Narrowing

WIDENING (Implicit - Safe, No data loss) byte -> short -> int -> long -> float -> double char --^ NARROWING (Explicit - Risky, Possible data loss) double -> float -> long -> int -> short -> byte 
// WIDENING: Automatic (small to big) - SAFE
int i = 100;
long l = i;        // int -> long (automatic)
double d = i;      // int -> double (automatic)

// NARROWING: Manual cast required (big to small) - RISKY
double d2 = 9.99;
int i2 = (int) d2;  // 9 (decimal part LOST!)

int big = 130;
byte b = (byte) big; // -126 (overflow! 130 wraps around in byte range)
⚠️ Common Mistakes
❌ long l = 10; float f = l; - Compiles but may lose precision! long has 64 bits, float has 32 bits.
❌ Assuming (int) rounds a double. It TRUNCATES (chops off decimal), doesn't round. Use Math.round() for rounding.

What is the var keyword (Java 10+)?

🤔 What is it?
var is a LOCAL variable type inference feature. The compiler figures out the type for you from the right side. It does NOT make Java dynamically typed - the type is still fixed at compile time. 
// Without var
Map<String, List<Integer>> map = new HashMap<String, List<Integer>>();

// With var (cleaner!)
var map = new HashMap<String, List<Integer>>();

// More examples
var name = "John";           // Inferred as String
var list = List.of(1,2,3);    // Inferred as List<Integer>
var stream = list.stream();   // Inferred as Stream<Integer>

What is a Package in Java?

🤔 What is it?
A package is like a FOLDER that groups related classes. It prevents naming conflicts and provides access control.
Analogy: Like organizing files in folders on your computer. You can have a file called Report.docx in both Sales/ and HR/ folders without conflict. 
// Declaring a package (MUST be first line)
package com.company.service;

// Importing from another package
import com.company.model.User;
import com.company.model.*;    // Import all classes (not sub-packages!)
import static java.lang.Math.PI; // Static import
⚠️ Common Mistakes
❌ import java.util.* does NOT import sub-packages like java.util.concurrent.*
❌ java.lang is auto-imported, you don't need to import String, System, etc.
❌ Wildcard imports (*) can cause ambiguity if two packages have same class name

Is Java Pass by Value or Pass by Reference? HOTTRICKY

🤔 The Answer
Java is ALWAYS Pass by Value. Period. But here's the confusion: when you pass an object, you're passing the VALUE of the reference (memory address), not the object itself. 
// PRIMITIVE: Pass by value (copy of value)
void change(int x) { x = 99; }

int a = 10;
change(a);
System.out.println(a);  // Still 10! (copy was changed, not original)

// OBJECT: Pass by value OF THE REFERENCE (copy of address)
void modify(StringBuilder sb) {
    sb.append(" World");  // This WORKS - modifying same object via copied reference
}

void reassign(StringBuilder sb) {
    sb = new StringBuilder("New");  // This does NOT affect original
}

StringBuilder s = new StringBuilder("Hello");
modify(s);
System.out.println(s);  // "Hello World" (object was modified)

reassign(s);
System.out.println(s);  // STILL "Hello World" (reference copy was reassigned, not original)
PASS BY VALUE OF REFERENCE: Calling Code: Method: s ----+ sb ----+ (sb is a COPY of s) | | v v [StringBuilder: "Hello"] (same object!) After sb.append(" World"): s ----+ sb ----+ | | v v [StringBuilder: "Hello World"] (modified via copied ref) After sb = new StringBuilder("New"): s ----+ sb ----+ | | v v [Hello World] [New] (sb now points elsewhere, s unchanged!)

What is the strictfp keyword?

🤔 What is it?
strictfp ensures that floating-point calculations give the SAME result on every platform. Without it, different CPUs may produce slightly different decimal results. 
strictfp class Calculator {
    double calculate(double a, double b) {
        return a * b;  // Guaranteed same result on ALL platforms
    }
}
// Note: Since Java 17, strictfp is effectively default. The keyword is optional.

What is the transient keyword?

🤔 What is it?
transient tells Java: "Don't save this field when serializing the object." Used for sensitive data like passwords, or fields that can be re-computed. 
public class User implements Serializable {
    String username = "john";
    transient String password = "secret123";  // Will NOT be saved
    transient int age = 25;                    // Will become 0 after deserialization
}

// After serialize + deserialize:
// username = "john"    (saved)
// password = null      (transient - not saved, gets default null)
// age = 0              (transient - not saved, gets default 0)

What is the volatile keyword? HOT

🤔 What is it?
volatile tells Java: "Always read this variable from MAIN MEMORY, not from a thread's local cache." It ensures visibility of changes across threads. 
class SharedFlag {
    volatile boolean running = true;
    // Without volatile, Thread 2 might never see Thread 1's change!
}

// Thread 1: flag.running = false;  (writes to main memory immediately)
// Thread 2: while(flag.running)    (reads from main memory, sees false)
⚠️ Important
volatile guarantees VISIBILITY but NOT ATOMICITY. volatile int count; count++; is NOT thread-safe because count++ is actually 3 operations (read, increment, write). Use AtomicInteger for that.

What is the instanceof operator?

// Checks if an object IS-A type

System.out.println(s instanceof Object);   // true (String extends Object)
System.out.println(null instanceof String); // false (null is not instance of anything)

// Java 16+: Pattern Matching instanceof (avoids extra cast!)
Object obj = "Hello";

// OLD way:
if (obj instanceof String) {
    String str = (String) obj;  // Manual cast needed
    System.out.println(str.length());
}

// NEW way (Java 16+):
if (obj instanceof String str) {  // Cast + assign in one step!

What are Annotations? MUST KNOW

🤔 What are they?
Annotations are METADATA (data about data) that you attach to code. They don't change the code behavior directly but provide instructions to the compiler, runtime, or tools (like Spring, JPA, etc.). 
// Built-in Annotations
@Override      // Tells compiler: "I'm overriding a parent method" (compile error if not)
@Deprecated   // Warns: "This is old, don't use it"
@SuppressWarnings("unchecked")  // Hides compiler warnings
@FunctionalInterface           // Ensures interface has exactly 1 abstract method
@SafeVarargs                    // Suppresses heap pollution warning

// Custom Annotation
@Retention(RetentionPolicy.RUNTIME)   // Available at runtime (for reflection)
@Target(ElementType.METHOD)             // Can only be used on methods
public @interface LogExecutionTime {
    String value() default "INFO";
}

// Usage
@LogExecutionTime("DEBUG")
public void processOrder() { /* ... */ }
🌐 Where used in real projects?
- Spring: @Autowired, @Service, @RestController, @Transactional
- JPA/Hibernate: @Entity, @Table, @Column, @OneToMany
- JUnit: @Test, @BeforeEach, @MockBean
- Lombok: @Getter, @Setter, @Builder
- Swagger: @ApiOperation, @ApiResponse

What is Reflection API? HOT

🤔 What is it?
Reflection lets you inspect and modify class structure at RUNTIME. You can access private fields, invoke private methods, create objects dynamically - all things you normally can't do.

Analogy: X-ray vision for Java classes. You can see inside any class, even private parts. 
class Secret {
    private String password = "hidden123";
    private void secretMethod() { System.out.println("Secret!"); }
}

// Using Reflection to access PRIVATE members
Secret obj = new Secret();
Class<?> clazz = obj.getClass();

// Access private field
Field field = clazz.getDeclaredField("password");
field.setAccessible(true);  // Bypass private!
System.out.println(field.get(obj));  // "hidden123"

// Call private method
Method method = clazz.getDeclaredMethod("secretMethod");
method.setAccessible(true);
method.invoke(obj);  // Prints: "Secret!"
🌐 Where is it used?
Spring Framework (Dependency Injection), Hibernate (ORM mapping), JUnit (test discovery), Jackson (JSON to Object), Serialization frameworks
⚠️ Downsides
Slow (bypasses JIT optimizations), breaks encapsulation, compile-time safety lost, may break with Java module system (Java 9+). Avoid in application code - let frameworks handle it.

What is Serialization and Deserialization? HOT

🤔 What is it?
Serialization = Converting an object to a byte stream (to save to file or send over network)
Deserialization = Converting byte stream back to an object

Analogy: Disassembling IKEA furniture to ship it (serialization), then reassembling at the destination (deserialization). 
public class Employee implements Serializable {
    private static final long serialVersionUID = 1L; // Version control!
    String name = "John";
    int age = 30;
    transient String password = "secret";  // Won't be serialized
}

// SERIALIZE (write object to file)
Employee emp = new Employee();
ObjectOutputStream oos = new ObjectOutputStream(
    new FileOutputStream("employee.ser"));
oos.writeObject(emp);
oos.close();

// DESERIALIZE (read object from file)
ObjectInputStream ois = new ObjectInputStream(
    new FileInputStream("employee.ser"));
Employee restored = (Employee) ois.readObject();
ois.close();
// restored.name = "John", restored.age = 30, restored.password = null
⚠️ Key Points
- serialVersionUID: If you don't declare it, JVM auto-generates one. If class changes, UID changes, and deserialization FAILS with InvalidClassException.
- static fields are NOT serialized (they belong to class, not object).
- transient fields are NOT serialized (get default values on deserialization).
- All fields' types must ALSO be Serializable, or you get NotSerializableException.

What is Object Cloning?

public class Person implements Cloneable {
    String name;
    Address address;  // reference type

    @Override
    protected Object clone() throws CloneNotSupportedException {
        return super.clone();  // SHALLOW copy
    }
}

Person p1 = new Person("John", new Address("NYC"));
Person p2 = (Person) p1.clone();

p2.name = "Jane";             // p1.name still "John" (String is immutable)
p2.address.city = "LA";        // p1.address.city ALSO becomes "LA"! (shared ref)
31

Shallow Copy vs Deep Copy HOT

SHALLOW COPY: p1 ---> [name: "John"] [address ----> [city: "NYC"]] p2 ---> [name: "John"] [address -----^ ] <-- SHARES same Address! DEEP COPY: p1 ---> [name: "John"] [address ----> [city: "NYC"]] p2 ---> [name: "John"] [address ----> [city: "NYC"]] <-- SEPARATE Address copy! 
// DEEP COPY: Override clone to clone inner objects too
@Override
protected Object clone() throws CloneNotSupportedException {
    Person cloned = (Person) super.clone();
    cloned.address = (Address) this.address.clone();  // Clone inner object!
    return cloned;
}

// Alternative: Use Copy Constructor (Recommended!)
public Person(Person other) {
    this.name = other.name;
    this.address = new Address(other.address);  // New object
}

// Alternative: Serialize and Deserialize (Deep copy any Serializable object)

What are Records (Java 16+)? MODERN

🤔 What is it?
A Record is a compact way to create immutable data classes. Java automatically generates constructor, getters, equals(), hashCode(), and toString(). 
// OLD way: 50+ lines of boilerplate!
public class Employee {
    private final String name;
    private final int age;
    // Constructor, getters, equals, hashCode, toString... SO MUCH CODE
}

// NEW way with Record: 1 LINE!
public record Employee(String name, int age) { }

// Usage
Employee e = new Employee("John", 30);
e.name();     // "John" (NOT getName()!)
e.age();      // 30
e.toString(); // "Employee[name=John, age=30]"

// Records can have:
public record Employee(String name, int age) {
    // Compact constructor (validation)
    public Employee {
        if (age < 0) throw new IllegalArgumentException("Age cannot be negative");
    }
    // Static methods, instance methods, implement interfaces
    public String greeting() { return "Hi, I'm " + name; }
}
⚠️ Limitations
- Records are implicitly final (can't be extended)
- Fields are implicitly private final (immutable)
- Can't extend other classes (already extends java.lang.Record)
- CAN implement interfaces

What are Sealed Classes (Java 17)? MODERN

🤔 What is it?
Sealed classes let you CONTROL which classes can extend/implement them. It's like an exclusive club with a guest list. 
// Only Circle, Rectangle, and Triangle can extend Shape
public sealed class Shape
    permits Circle, Rectangle, Triangle {
}

public final class Circle extends Shape { }       // final = no more subclasses
public sealed class Rectangle extends Shape
    permits Square { }                               // sealed = controlled extension
public non-sealed class Triangle extends Shape { } // non-sealed = anyone can extend

// WHY? Enables exhaustive pattern matching in switch:
String describe(Shape shape) {
    return switch (shape) {
        case Circle c    -> "Circle";
        case Rectangle r -> "Rectangle";
        case Triangle t  -> "Triangle";
        // No default needed! Compiler knows all subtypes.
    };
}

Pattern Matching for instanceof (Java 16+)

// OLD: Check type + cast = 2 steps
if (obj instanceof String) {
    String s = (String) obj;
    System.out.println(s.toUpperCase());
}

// NEW: Check type + cast + assign = 1 step!
if (obj instanceof String s) {
    System.out.println(s.toUpperCase());  // s is already cast!
}

// Works with negation too:
if (!(obj instanceof String s)) {
    return;  // early return
}
System.out.println(s.toUpperCase());  // s is in scope here!

Switch Expressions (Java 14+) MODERN

// OLD switch (statement)
int numLetters;
switch (day) {
    case MONDAY: case FRIDAY:
        numLetters = 6;
        break;     // Don't forget break! (bug source)
    case TUESDAY:
        numLetters = 7;
        break;
    default:
        numLetters = 0;
}

// NEW switch EXPRESSION (Java 14+)
int numLetters = switch (day) {
    case MONDAY, FRIDAY  -> 6;    // No break needed! Arrow syntax.
    case TUESDAY         -> 7;
    case WEDNESDAY       -> 9;
    default              -> 0;
};  // Note the semicolon! It's an expression now.

// Multi-line blocks use yield
int result = switch (status) {
    case OK -> 1;
    case ERROR -> {
        log("Error occurred");
        yield -1;  // yield returns value from block
    }
    default -> 0;
};

Garbage Collection Overview

🤔 What is it?
GC is the automatic memory management system. It finds objects that are no longer reachable (no references pointing to them) and frees their memory. You DON'T manually free memory in Java. 
Employee e1 = new Employee();  // Object 1 created on heap
Employee e2 = new Employee();  // Object 2 created on heap
e1 = null;                      // Object 1 now ELIGIBLE for GC
e1 = e2;                         // (if e1 was pointing to something, that becomes eligible)

// You can REQUEST GC (but JVM may ignore it!)
System.gc();       // Just a suggestion, not a command!
Runtime.getRuntime().gc();  // Same thing
⚠️ Key Points
- You can NEVER force GC. System.gc() is just a hint.
- finalize() is deprecated since Java 9. Don't use it.
- More details in the Memory Management FAQ file.

equals() vs == HOTTRICKY

🤔 The Core Rule
== compares REFERENCES (are they the same object in memory?)
.equals() compares VALUES/CONTENT (are the contents the same?) 
String a = new String("Hello");
String b = new String("Hello");

a == b          // FALSE (different objects in heap)
a.equals(b)    // TRUE  (same content "Hello")

// TRICKY: String pool
String c = "Hello";
String d = "Hello";
c == d          // TRUE! (same object in String pool)
c.equals(d)    // TRUE  (same content)

// PRIMITIVES: == compares VALUES (no .equals() available)
int x = 5, y = 5;
x == y          // TRUE (comparing values directly)
Type== (double equals).equals()
PrimitivesCompares valuesNot available
ObjectsCompares references (memory address)Compares content (if properly overridden)
String literalsMay be true (pool)Always true for same content
new String()Always false (diff objects)True for same content
✅ Best Practice
ALWAYS use .equals() for object comparison. Use == only for primitives and null checks. For null-safe comparison use Objects.equals(a, b).

hashCode() and its contract with equals() HOT

🤔 What is it?
hashCode() returns an integer that represents the object. It's used by hash-based collections like HashMap, HashSet to determine which "bucket" to store the object in.
THE CONTRACT (you MUST follow this):
RuleExplanation
If a.equals(b) is TRUEThen a.hashCode() MUST equal b.hashCode()
If a.hashCode() == b.hashCode()a.equals(b) MAY or MAY NOT be true (collision allowed)
If a.equals(b) is FALSEhashCodes CAN be same (but shouldn't be, for performance)
public class Employee {
    String id;
    String name;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Employee emp = (Employee) o;
        return Objects.equals(id, emp.id);
    }

    @Override
    public int hashCode() {
        return Objects.hash(id);  // MUST use same fields as equals()!
    }
}
⚠️ What happens if you break the contract?
If you override equals() but NOT hashCode(), HashMap/HashSet will NOT work correctly. Two "equal" objects could end up in different buckets, and you'll never find them.

Rule of thumb: If you override equals(), ALWAYS override hashCode() too.
