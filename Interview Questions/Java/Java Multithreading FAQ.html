Java Memory Model (JMM) - happens-before HOT

// JMM defines when one thread's write is VISIBLE to another thread's read
// "happens-before" = guarantee of visibility

// These establish happens-before:
// 1. synchronized exit -> next synchronized entry (same lock)
// 2. volatile write -> next volatile read (same var)
// 3. Thread.start() -> everything in run()
// 4. Everything in run() -> Thread.join() returns
// 5. Final field write in constructor -> any read after constructor

volatile boolean ready = false;
int data = 0;
// Thread A:
data = 42; ready = true;       // volatile write
// Thread B:
if (ready) print(data);        // GUARANTEED to see 42
Without guarantee: With volatile/sync: Thread A: data=42, ready=true Thread A: data=42, ready=true Thread B: ready=true, data=0 Thread B: ready=true, data=42 STALE! CORRECT!

InterruptedException handling HOT

// WRONG: swallowing interrupt
try { Thread.sleep(1000); }
catch (InterruptedException e) { } // BAD!

// RIGHT: re-interrupt (restore flag)
try { Thread.sleep(1000); }
catch (InterruptedException e) {
    Thread.currentThread().interrupt(); // restore flag
    return;
}

// RIGHT: propagate
void process() throws InterruptedException {
    Thread.sleep(1000);
}

// Non-blocking check:
while (!Thread.currentThread().isInterrupted()) { doWork(); }

Race Condition Example and Fix HOT

class BankAccount {
    private int balance = 1000;

    // WRONG: check-then-act NOT atomic!
    void withdraw(int amt) {
        if (balance >= amt) { // T1: 1000>=800 YES
            // T2 also checks: 1000>=800 YES (interleaved!)
            balance -= amt;    // T1: 200, T2: 200-800 = -600! BUG
        }
    }

    // FIX: synchronized
    synchronized void withdraw(int amt) {
        if (balance >= amt) balance -= amt;
    }
}
