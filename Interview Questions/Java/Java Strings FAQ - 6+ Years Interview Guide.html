Why are Strings Immutable? HOT

What does Immutable mean?
Once created, content CANNOT change. Every modification creates a NEW String.
String s = "Hello";
s.concat(" World");
System.out.println(s);  // Still "Hello"! concat returns NEW string
s = s.concat(" World"); // NOW s points to new "Hello World"
#Why Immutable?Explanation
1String PoolMultiple refs share same object. Mutation would affect all.
2SecurityDB URLs, passwords. Can't change after validation.
3Thread SafetyImmutable = inherently thread-safe.
4HashCode CachingComputed once, cached. HashMap depends on this.
5Class LoadingClass names are Strings. Prevents malicious changes.
6PerformancePool reuse saves memory. hashCode caching saves CPU.

StringTokenizer vs split() vs Pattern

String csv = "John,25,Developer,NYC";

// split() - most common (uses regex internally)
String[] parts = csv.split(",");  // ["John", "25", "Developer", "NYC"]

// StringTokenizer - legacy, faster for simple delimiters (no regex)
StringTokenizer st = new StringTokenizer(csv, ",");
while (st.hasMoreTokens()) {
    System.out.println(st.nextToken());
}

// Pattern - best for repeated splits (compile once, use many)
Pattern pattern = Pattern.compile(",");
String[] parts2 = pattern.split(csv); // Reuse compiled pattern

// Performance: Pattern (pre-compiled) > StringTokenizer > split()
// split() compiles regex EVERY time it's called

String Deduplication (JVM) Advanced

// JVM G1 GC has automatic String Deduplication
// Enable: -XX:+UseG1GC -XX:+UseStringDeduplication
// (default on since Java 9 with G1)

// What it does:
// If two String objects have same char[] content,
// GC makes them share the SAME char[] array
// Saves memory without changing object identity

// Different from intern():
// intern() shares String OBJECT (from pool)
// Dedup shares internal char[] ARRAY (transparent to code)

// When useful: Large apps with many duplicate strings
// e.g., city names, country codes, status values

Compact Strings (Java 9+)

// Before Java 9: String used char[] (2 bytes per char, UTF-16)
// After Java 9: String uses byte[] + coder flag
// Latin1 chars (ASCII): 1 byte per char (LATIN1 encoding)
// Non-Latin chars: 2 bytes per char (UTF-16 encoding)

// Result: ~50% memory savings for ASCII-heavy apps (most English apps)
// Enabled by default. Disable: -XX:-CompactStrings

// You don't need to change any code - it's transparent!

Why use char[] for passwords instead of String? HOT

// String: immutable, stays in memory until GC, visible in heap dumps
// char[]: mutable, YOU can clear it after use!

void authenticate(char[] password) {
    try {
        boolean valid = checkPassword(password);
    } finally {
        Arrays.fill(password, '0'); // Wipe from memory immediately!
    }
}

String strip() vs trim() (Java 11+)

// trim() - removes chars <= ' ' (space, ASCII 0-32)
// strip() - removes Unicode whitespace (more thorough)

String s = "\u2000 Hello \u2000"; // \u2000 = EN QUAD (Unicode space)
s.trim();    // "\u2000 Hello \u2000" - NOT removed!
s.strip();   // "Hello" - Unicode space REMOVED!

s.stripLeading();   // "Hello \u2000"
s.stripTrailing();  // "\u2000 Hello"

// Rule: Always use strip() (Java 11+) instead of trim()
// trim() only knows about ASCII whitespace

Common String Interview Coding Problems HOT

// 1. Reverse a String

// 2. Check palindrome
boolean isPalindrome(String s) {
    String clean = s.replaceAll("[^a-zA-Z0-9]", "").toLowerCase();
    return clean.equals(new StringBuilder(clean).reverse().toString());
}

// 3. Count character frequency
Map<Character, Long> freq = str.chars()
    .mapToObj(c -> (char) c)
    .collect(Collectors.groupingBy(c -> c, Collectors.counting()));

// 4. Find first non-repeating character
char first = str.chars()
    .mapToObj(c -> (char) c)
    .collect(Collectors.groupingBy(c -> c, LinkedHashMap::new, Collectors.counting()))
    .entrySet().stream()
    .filter(e -> e.getValue() == 1)
    .map(Map.Entry::getKey)
    .findFirst().orElse('\0');

// 5. Check anagram
boolean isAnagram(String a, String b) {
    char[] x = a.toLowerCase().toCharArray(); Arrays.sort(x);
    char[] y = b.toLowerCase().toCharArray(); Arrays.sort(y);
    return Arrays.equals(x, y);
}

// 6. Remove duplicates

String Interning Deep Dive HOT

// intern() puts string in pool and returns pooled reference
String s1 = new String("Hello");
String s2 = s1.intern(); // Returns reference from pool
String s3 = "Hello";     // Already in pool

System.out.println(s1 == s2); // false (s1 is heap, s2 is pool)
System.out.println(s2 == s3); // true  (both pool references)

// When to use intern():
// - Processing millions of strings with many duplicates
// - Parsing CSV/XML with repeated values
// Caution: intern() uses native memory, can cause OOM if overused
// Java 7+: intern pool is on heap, GC'd normally
// Resize pool: -XX:StringTableSize=60013 (prime number)
